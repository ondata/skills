# Notes

This folder contains internal working notes for the onData skills team.

These notes are **not part of any skill** and are never installed alongside skill files. They exist solely to support the people who build, maintain, and evaluate skills in this repository.

Notes can capture anything useful that doesn't belong in the skill itself: background decisions, known limitations, pitfalls discovered during testing, ideas for future work, or useful external references.

## How to use

Create one Markdown file per skill, named after the skill (matching the `name` field in its `SKILL.md` frontmatter). Example: `notes/openalex.md` for the `openalex` skill.

## Template (optional)

The structure below is a suggestion, not a requirement. Add only the sections that are useful.

```markdown
# <skill-name>

## Context
Why this skill exists, relevant background, key design decisions.

## Known limitations
Things that don't work well or were intentionally left out.

## Ideas & future work
Improvements, experiments, or features worth exploring later.

## Pitfalls & gotchas
Traps found during development or evaluation runs.

## Links
Useful external resources (API docs, articles, datasets, etc.).
```
